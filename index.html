Caveats

n = 1. A single bear clip; no statistical claim. The metric gaps shown are illustrative, not statistically significant.

Prompt iteration matters. The original v3 suffix ("no drastic camera movement … only small and smooth movement") was permissive enough that Sora applied a slow dolly. The current suffix ("no camera large motions are allowed and keep the camera fairly static") locks the camera, which raised Sora's Tem-Con +0.017 and Subj-Cons +0.011 on this clip.

Sora pacing. Sora generated 4 s of bear walking; we trimmed to the first 2.04 s (round(D_target × sora_fps)). The trimmed action reflects Sora's pacing across the full 4 s — if motion is back-loaded we capture only setup motion. Eyeball the hero clip to confirm.

Sora 2 is 30 fps, not 24 as some docs claim. We probe the actual fps from the downloaded mp4 via cv2 and use it for the trim count.

DaS prompt asymmetry. The DaS bear smoke (existing reference) was generated 2026-04-28 with the old suffix; only Sora was rerun on the new suffix. A future DaS rerun under the new suffix would close this asymmetry.
