Caveats

n = 1. A single bear clip; no statistical claim. The 0.0001 gap on Subj-Cons between DaS and Sora is noise; only Tem-Con shows a meaningful gap.

Sora pacing. Sora generated 4 s of bear walking; we trimmed to the first 2.04 s. The trimmed action reflects whatever pacing Sora chose across the full 4 s — if motion was back-loaded, we capture only setup motion. Eyeball the comparison to confirm.

Sora 2 is 30 fps, not 24 as some docs claim. We probe the actual fps from the downloaded mp4 via cv2 and use it for the trim count (round(D_target × sora_fps)).
